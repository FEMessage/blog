# 前言

以上传组件在github上一个pr为例，演示描述如何从通过金字塔原理，由初始的版本演化成结构分明、逻辑清晰的最终版本。

# 版本1

思路：说明一下各种操作的表现，这样用户使用时能提前了解，知道组件能力的边界，而不会认为是“怪异”行为

不足：红框内部应该是兼容性说明，而不是Fixes的内容

# 版本2

思路：

  • 兼容性说明独立放到 compatibility

不足：

  • draggable 与 android select or take photos 是两件事，不应该放一起, 后者应该是另一个pr的内容
  • 同时兼容性说明不够清晰完整，没有提到全部情况

# 版本3


思路：上面考虑到了以下情况

  • 粘贴/拖拽
  • 单张模式/多张模式
  • Windows系统/Mac系统


但依然存在不足：

  • 依然存在无法粘贴的问题 v1.20，是指什么
  • 其实 粘贴 跟 多张模式 无关，因为 Mac系统下 copy 多张，实际只能 paste 上传一张

# 版本4


思路：

  • 把 粘贴 与 拖拽 放到外层，单张模式/多张模式 放在内层
  • 详细描述 粘贴 在 Windows 系统下的兼容情况

发现还是存在不足：

  • 拖拽与粘贴文字出现的顺序，Mac系统/Windows系统不一致
  • Mac下的粘贴说明有点费解

# 版本5


跟版本1比起来，是不是清晰明了很多！
